# Security Policy

## Supported Versions
| Version | Supported |
|---|---|
| 0.1.x | ✅ |

## Reporting a Vulnerability
If you discover a security vulnerability in GhostLM, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, email: joemunene984@gmail.com with subject line: [GhostLM SECURITY]

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

You will receive a response within 48 hours.

## Responsible Use
GhostLM is built for cybersecurity research and education. Users are expected to:
- Only use GhostLM on systems they own or have explicit permission to test
- Not use GhostLM to generate malware or exploit code for malicious purposes
- Comply with all applicable laws and regulations
- Follow responsible disclosure practices

## Scope
This policy covers the GhostLM codebase at github.com/joemunene-by/GhostLM.
Model weights and generated outputs are the responsibility of the user.
